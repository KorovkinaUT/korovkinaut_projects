# Link Tracker

Link Tracker — это приложение для отслеживания обновлений по ссылкам и отправки уведомлений в Telegram. Проект состоит из Telegram-бота и сервиса, который хранит отслеживаемые ссылки, периодически проверяет, не появлись ли какие-то обновления, и отправляет уведомления об обновлениях пользователю.

На данный момент проект ещё разрабатывается, реализованный функционал не полный.


## Что уже реализовано:

### 1. Telegram-бот
Поддерживаемые команды:

| Команда | Описание |
|-------|--------|
| `/start` | регистрация пользователя в системе |
| `/help` | вывод списка доступных команд |
| `/track` | добавить ссылку для отслеживания, поддерживаются теги для ссылок |
| `/untrack` | удалить ссылку из отслеживания |
| `/list` | показать отслеживаемые ссылки |

Команды подключаются через набор handlers, а маршрутизация сообщений сделана через dispatcher.

### 2. Диалог добавления ссылки
Поддерживается добавление ссылки для отслеживания в формате диалога:
1. пользователь вводит команду `/track`
2. бот переводит пользователя в состояние ожидания ссылки
3. пользователь отправляет URL
4. бот переводит пользователя в состояние ожидания тегов для ссылки
5. пользователь вводит теги или `/cancel` для добавления без тегов
6. ссылка передаётся в scrapper-сервис
7. бот завершает сценарий

Для этого используется отдельная сущность состояния пользователя, чтобы помнить, на каком шаге диалога находится конкретный chat id.

### 4. Управление ссылками

Бот взаимодействует с **scrapper-сервисом** через HTTP API.

Поддерживаемые операции:

- регистрация чата
- добавление ссылки
- удаление ссылки
- получение списка ссылок

### 5. Проверка обновлений

В системе реализован компонент **checker / scheduler**, который:

1. получает список отслеживаемых ссылок
2. определяет тип ссылки
3. вызывает соответствующий клиент
4. проверяет наличие изменений
5. отправляет обновление боту

Архитектура checker разработана так, чтобы легко добавлять новые типы источников.

На текущем этапе реализовано отслеживание обновлений с сайтов через HTTP:

- **GitHub**
- **Stack Overflow**

В дальнейшем планируется поддержка дополнительных сервисов.

### 6. Хранение данных

На данный момент не подключена база данных, используется in-memory хранение и паттер Repository.

### 7. Тесты

Тесты покрывают базовую бизнес-логику и HTTP-взаимодействие. Используются mock-тесты.

## Инструкция по запуску

### .env файл
Перед запуском необходимо создать файл `.env` в корне проекта (пример есть в .env.example).

Минимально необходимая переменная окружения:

```env
APP_TELEGRAM_TOKEN=your_token
````

Остальные параметры можно не указывать — будут использованы значения по умолчанию:

Конфигурация бота:

| Переменная             | Описание                      | Значение по умолчанию       |
| ---------------------- | ----------------------------- | --------------------------- |
| `APP_TELEGRAM_TOKEN`   | Токен Telegram бота           | **обязательная переменная** |
| `POLL_TIMEOUT_SECONDS` | Таймаут long polling Telegram | `30`                        |
| `BOT_HOST`             | Хост HTTP-сервера бота        | `localhost`                 |
| `BOT_PORT`             | Порт HTTP-сервера бота        | `8080`                      |
| `SCRAPPER_HOST`        | Хост сервиса scrapper         | `localhost`                 |
| `SCRAPPER_PORT`        | Порт сервиса scrapper         | `8081`                      |
| `HTTP_TIMEOUT`         | Таймаут HTTP-клиентов         | `5s`                        |

Конфигурация scrapper:

| Переменная               | Описание                     | Значение по умолчанию               |
| ------------------------ | ---------------------------- | ----------------------------------- |
| `SCHEDULER_INTERVAL`     | Интервал проверки обновлений | `1m`                                |
| `BOT_HOST`               | Хост сервиса бота            | `localhost`                         |
| `BOT_PORT`               | Порт сервиса бота            | `8080`                              |
| `SCRAPPER_HOST`          | Хост HTTP-сервера scrapper   | `localhost`                         |
| `SCRAPPER_PORT`          | Порт HTTP-сервера scrapper   | `8081`                              |
| `HTTP_TIMEOUT`           | Таймаут HTTP-клиентов        | `5s`

### Запуск сервисов

Для запуска всего приложения введите `make run` из корня репозитория.

Для запуска только бота: `make bot`

для запуска только scrapper: `make scrapper`

### Запуск тестов

Из корня репозитория: `go test ./...`
