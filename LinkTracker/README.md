# Link Tracker

Link Tracker — это приложение для отслеживания обновлений по ссылкам и отправки уведомлений в Telegram. Проект состоит из Telegram-бота и сервиса, который хранит отслеживаемые ссылки, периодически проверяет, не появлись ли какие-то обновления, и отправляет уведомления об обновлениях пользователю.


## Реализация:

### 1. Telegram-бот
Поддерживаемые команды:

| Команда | Описание |
|-------|--------|
| `/start` | регистрация пользователя в системе |
| `/help` | вывод списка доступных команд |
| `/track` | добавить ссылку для отслеживания, поддерживаются теги для ссылок |
| `/untrack` | удалить ссылку из отслеживания |
| `/list` | показать отслеживаемые ссылки |

Команды подключаются через набор handlers, а маршрутизация сообщений сделана через dispatcher.

### 2. Диалог добавления ссылки
Поддерживается добавление ссылки для отслеживания в формате диалога:
1. пользователь вводит команду `/track`
2. бот переводит пользователя в состояние ожидания ссылки
3. пользователь отправляет URL
4. бот переводит пользователя в состояние ожидания тегов для ссылки
5. пользователь вводит теги или `/cancel` для добавления без тегов
6. ссылка передаётся в scrapper-сервис
7. бот завершает сценарий

Для этого используется отдельная сущность состояния пользователя, чтобы помнить, на каком шаге диалога находится конкретный chat id.

### 4. Управление ссылками

Бот взаимодействует с **scrapper-сервисом** через HTTP API.

Поддерживаемые операции:

- регистрация чата
- добавление ссылки
- удаление ссылки
- получение списка ссылок

### 5. Проверка обновлений

Реализован компонент **checker**, который отвечает за периодическую проверку обновлений ссылок. Он является отдельным сервисом, запускаемый по расписанию через **scheduler**. **Checker** проверяет ссылки батчами и поддерживает параллельную проверку обновлений.

Общий алгоритм работы **checker**:

1. получить батч отслеживаемых ссылок
2. Запустить worker-ов для проверки обновлений
3. Положить ссылки в канал-очередь
4. Для каждой ссылки:

   * определить тип (GitHub / StackOverflow)
   * получить новые события через соответствующий клиент
   * отфильтровать события по `last_updated_at`
5. Если есть новые события:

   * сформировать сообщение
   * отправить его в бот
   * обновить `last_updated_at` в базе
6. Сформировать сообщение с описнаием ошибок по ссылкам, которые не удалось обработать и отправить его в бот

Реализовано отслеживание обновлений с сайтов через HTTP:

- **GitHub**
- **Stack Overflow**

Архитектура построена на интерфейсах, что позволяет легко добавить отслеживание других ссылок с сайтов.


### 6. Хранение данных

Данные хранятся в базе данных **PostgreSQL**.

Используется следующая схема:

* `chats` — зарегистрированные пользователи
* `links` — отслеживаемые ссылки
* `link_chat` — подписки чатов на ссылки
* `link_tag` — теги ссылок, заданные пользователем

Одна ссылка хранится один раз независимо от количества пользователей, поддерживается каскадное удаление и для оптимизации используются индексы.

Доступ к базе данных реализован через паттерн **Repository**.

Поддерживаются две реализации (настривается через конфигурацию):

* `SQL` — работа через raw SQL
* `GOQU` — работа через query builder

Фабрика репозиториев создаёт нужную реализацию в зависимости от конфигурации.

Все операции с ссылками (`AddLink`, `RemoveLink`) выполняются в рамках транзакций.

### 7. Тесты

Реализованы unit-тесты для проверки работы основных компонент системы. И интеграционные тесты для проверки совместной работы различных компонент.

## Инструкция по запуску

### .env файл
Перед запуском необходимо создать файл `.env` в корне проекта (пример есть в .env.example).

Минимально необходимая переменная окружения:

```env
APP_TELEGRAM_TOKEN=your_token
````

Остальные параметры можно не указывать — будут использованы значения по умолчанию:

Общие перменные:
| Переменная         | Описание                  | Значение по умолчанию |
| ------------------ | ------------------------- | --------------------- |
| `HTTP_TIMEOUT`     | Таймаут HTTP-клиентов     | `5s`                  |
| `SHUTDOWN_TIMEOUT` | Таймаут graceful shutdown | `10s`                 |

Конфигурация бота:

| Переменная             | Описание               | Значение по умолчанию |
| ---------------------- | ---------------------- | --------------------- |
| `APP_TELEGRAM_TOKEN`   | Токен Telegram бота    | **обязательная**      |
| `POLL_TIMEOUT_SECONDS` | Таймаут long polling   | `30`                  |
| `BOT_HOST`             | Хост HTTP-сервера бота | `localhost`           |
| `BOT_PORT`             | Порт HTTP-сервера бота | `8080`                |
| `SCRAPPER_HOST`        | Хост scrapper-сервиса  | `localhost`           |
| `SCRAPPER_PORT`        | Порт scrapper-сервиса  | `8081`                |

Конфигурация scrapper:

| Переменная           | Описание                     | Значение по умолчанию |
| -------------------- | ---------------------------- | --------------------- |
| `SCHEDULER_INTERVAL` | Интервал проверки обновлений | `1m`                  |
| `BATCH_SIZE`         | Размер батча ссылок для проверки обновлений| `100`             |
| `WORKERS_COUNT`      | Количество параллельных потоков для обработки обновлений|`4`|
| `SCRAPPER_HOST`      | Хост HTTP-сервера scrapper   | `localhost`           |
| `SCRAPPER_PORT`      | Порт HTTP-сервера scrapper   | `8081`                |
| `BOT_HOST`           | Хост сервиса бота            | `localhost`           |
| `BOT_PORT`           | Порт сервиса бота            | `8080`                |
| `GITHUB_BASE_URL`        | API GitHub        | `https://api.github.com`            |
| `STACKOVERFLOW_BASE_URL` | API StackOverflow | `https://api.stackexchange.com/2.3` |

Конфигурация базы данных:

| Переменная           | Описание            | Значение по умолчанию |
| -------------------- | ------------------- | --------------------- |
| `POSTGRES_DB`        | Имя базы            | `link_tracker`        |
| `POSTGRES_USER`      | Пользователь        | `postgres`            |
| `POSTGRES_PASSWORD`  | Пароль              | `postgres`            |
| `POSTGRES_HOST`      | Хост базы           | `localhost`           |
| `POSTGRES_PORT`      | Порт базы           | `5432`                |
| `POSTGRES_SSLMODE`   | SSL режим           | `disable`             |
| `DB_CONNECT_TIMEOUT` | Таймаут подключения | `5s`                  |
| `DB_ACCESS_TYPE`     | Тип доступа к БД ("SQL"/"GOCU")   | `SQL`                 |
